# Marp Style

Marp Markdown を書く / レビューする際の構造スタイル規約。

## スコープ

- 対象は Marp Markdown の構造 (見出し階層・スライド構成・各要素の書式)
- 対象外: 変換ツール (marp-cli / marp-vscode / 自作変換スクリプト等) 固有の挙動や PPTX レイアウト結果
- 対象外: 文体 (体言止め / です・ます調 / だ・である調 等)。既存原稿を確認する際も原文の文体は維持する。文体変換が必要な場合は別タスクで明示的に依頼する

## 必須ルール

このスキルの背骨。Marp 原稿を書く時もレビューする時も、まずこの 2 つを満たしているかを確認する。

### ルール 1: `#` (h1) はドキュメント全体で 1 つだけ

`#` 見出しは 1 枚目の表紙タイトル専用。2 枚目以降のスライドのタイトルは必ず `##` から始める。

### ルール 2: 表紙以外のすべてのスライドに `##` と `###` を両方書く

| 見出し | 役割 |
|---|---|
| `##` | そのスライドのテーマ (見出し) |
| `###` | そのスライドで一番言いたいこと (メインメッセージ) |

書き方:

- 各スライドに `###` を 1 つ以上書く (メインメッセージのないスライドを作らない)
- 配布資料として通読する想定なので、各スライドが 1 行のメインメッセージを持つことで読み手が要点を拾える
- 文体は本スキルの範囲外。原稿の文体には触らない

### レビュー時のチェック観点

- ドキュメント全体で `#` が 2 個以上ないか
- 表紙以外の各スライドに `##` と `###` がペアで揃っているか

## 全体構造

```markdown
---
marp: true
theme: default
paginate: true
header: '...'
footer: '...'
---

# 表紙タイトル

### 表紙のサブタイトル (任意)

本文 (任意)

---

## セクション 1 のタイトル

### このスライドのメインメッセージ

本文段落

- 箇条書き
- 箇条書き

---

## セクション 2 のタイトル

### このスライドのメインメッセージ

| 列 1 | 列 2 |
|---|---|
| a | b |

![代替テキスト](images/foo.png)
```

## スライド区切り

CommonMark の水平線 `---` でスライドが分割される。

- frontmatter の `---` はファイル先頭にある場合のみ frontmatter として認識される。それ以外の `---` は水平線扱い
- `***` や `___` も水平線になるが、慣例として `---` を使う

## 見出しレベル詳細

| Markdown | 役割 |
|---|---|
| `# 表題` | 表紙のタイトル (ドキュメント内で 1 回のみ) |
| `## 見出し` | スライドのタイトル (テーマ) |
| `### サブ` | スライドのメインメッセージ (1 スライドに 1 つ) |

- `####` 以下は本スタイルでは使わない
- 表紙 (`#` のスライド) は `###` を書いても書かなくてもよい

## 本文 (段落)

`##` / `###` 以外の通常段落はそのままスライドの本文として扱う。

- 段落間は空行で区切る
- 1 行の改行 (softbreak) は CommonMark 標準として 1 つの段落内の改行扱い

## リスト (箇条書き / 番号付き)

```markdown
- 親項目
  - 子項目 (半角スペース 2 個でネスト)
    - 孫項目 (さらに 2 個)

1. 番号付きリスト
1. 項目
1. ネストも同様に動く
```

ルール:

- 箇条書きは `-` または `*` を使う
- ネストは半角スペース 2 個または 4 個でインデント。タブは避ける
- リストとプレーン段落は混在可能。出現順に保持される

## GFM テーブル

```markdown
| 列 1 | 列 2 |
|---|---|
| a | b |
| c | d |
```

- ヘッダ行とセパレータ行 (`|---|---|`) を必ず書く
- セル内のインライン装飾の表示は変換ツール側の挙動に依存するため、ここでは規定しない

## 画像

```markdown
![代替テキスト](images/foo.png)
```

- 相対パスの基点は入力 Markdown ファイルの親ディレクトリ
- `images/` のようにサブディレクトリを切って整理する運用が無難
- リモート画像 (`http://` / `https://` / `data:`) のサポート可否は変換ツール依存のため、本スキルでは規定しない

## YAML frontmatter

```yaml
---
marp: true
theme: default
paginate: true
header: '...'
footer: '...'
---
```

- ファイル先頭の `---\n...\n---\n` のみが frontmatter として認識される
- `marp: true` で Marp として扱われる
- `theme` / `paginate` / `header` / `footer` は Marp 標準のディレクティブ
- これらディレクティブが実際にどう反映されるかは変換ツール側の責務

## HTML コメント

`<!-- ... -->` は Marp のレンダリング対象外として扱われる。

- markdownlint 抑制コメント (`<!-- markdownlint-disable MD013 -->` 等) をそのまま残せる
- 原稿内のメモとしても使える
