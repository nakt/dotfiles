# README テンプレート

`update-readme` スキルが生成・更新する README.md の章構成。見出しは英語で出力する。
各見出しの HTML コメントは判定基準と書き方の指針であり、生成後の README には残さない。

必須セクションは常に含める。条件付きセクションは検出条件を満たす場合のみ追加する。
コードを伴わないリポジトリ（設定リポジトリ・ドキュメント集など）では、見出しを実態に
合わせて読み替えてよい（例: Tech Stack → Contents）。

## スケルトン

```markdown
# <Project Name>

## Overview

<!-- 必須: 1-2 段落でプロジェクトの目的と概要 -->

## Tech Stack

<!-- 必須: 検出した言語・主要依存・ツール。コードを伴わないリポジトリでは "Contents" に読み替え可 -->

## Directory Structure

<!-- 必須: 主要なディレクトリとその役割 -->

## Setup

<!-- 必須: インストール・初期化手順 -->

## Usage

<!-- 必須: 基本的な使い方 -->

## Features

<!-- 条件付き: 機能や主な内容が明確な場合。アプリ以外では主な内容の箇条書きでも可 -->

## Development

<!-- 条件付き: dev dependencies がある場合の開発環境セットアップ -->

## Testing

<!-- 条件付き: tests/ ディレクトリがある場合のテスト実行方法 -->

## CLI Usage

<!-- 条件付き: コマンドラインツールの場合 -->

## API

<!-- 条件付き: Web フレームワークが検出された場合の API ドキュメント -->

## License

<!-- 条件付き: ライセンスファイルがある場合 -->
```
