# Markdown 作成ルール

## 基本方針
- 絵文字はユーザーが明示的に要求した場合を除き使用しない
- 強調（`**`）の多用を避ける

## 見出し構造
見出しを適切に使い、文章を正確に構造化する

### 良い例
```markdown
# タイトル

## セクション

### サブセクション
```

### 悪い例
```markdown
# タイトル
# セクション
# サブセクション
```

## 強調の使い方
本当に強調が必要な箇所のみに使用する

### 良い例
```markdown
この機能は`config.json`で設定できます。
```

### 悪い例
```markdown
**この**機能は**config.json**で**設定**できます。
```
