## Markdown ファイル作成時のスタイルガイド

### 基本方針

- 絵文字はユーザーが明示的に要求した場合を除き使用しない
- 強調（`**`）の多用を避ける

### 見出し構造

- 見出しを適切に使い、文章を正確に構造化する

良い例

```markdown
# タイトル

## セクション

### サブセクション
```

悪い例

```markdown
# タイトル
# セクション
# サブセクション
```

### 強調の使い方

- 本当に強調が必要な箇所のみに使用する

良い例

```markdown
この機能は`config.json`で設定できます。
```

悪い例

```markdown
**この**機能は**config.json**で**設定**できます。
```

## .workspace ディレクトリ運用ルール

### 基本方針

- 検証・一時実行・スクリプト実行結果などのファイルはプロジェクトルートの `.workspace/` に保存する
- `/tmp` や他の一時ディレクトリを使用しない

### ディレクトリ構造

- `.workspace/` 直下にファイルを置かず、作業単位でサブディレクトリを作成する
- ディレクトリ命名規則: `NN_作業名/`（例: `01_api-test/`, `02_migration-check/`）
- 数値プレフィックスは既存の最大値 + 1 を使う（なければ 01 から開始）

### バージョン管理

- `.workspace/` はバージョン管理対象外（`.gitignore` に追加済み）
